2. Put the site online — the easy way No coding

The fastest way to get a public link, using the free service Netlify Drop.

  1. Unzip the delivered ZIP (see section 1). Find the build folder.
  2. Open a web browser and go to https://app.netlify.com/drop
  3. Drag the whole build folder onto the page (the area that says “Drag and drop your site folder here”).
  4. Wait a few seconds. Netlify shows a live link like https://random-name.netlify.app — your site is online! 🎉
  5. (Optional) Create a free Netlify account to keep the site and rename the link.
  6. (Optional) Connect your own domain: in Netlify, “Domain settings” → “Add a domain”, then follow the on-screen steps.
Tip: Other free static hosts work the same way — Cloudflare Pages, Vercel, GitHub Pages, etc. Whatever you choose, you are simply publishing the contents of the build folder.
Please don't just double-click build/index.html to open it from your computer — some styling/links won't load that way. Always publish it to a host (above), or use the local-preview method in section 5.

3. Put it on your own server / hosting No coding

If you already have web hosting (a domain with cPanel, an FTP account, etc.):

  1. Log in to your hosting control panel (or open your FTP app, e.g. FileZilla).
  2. Open your website's public folder — usually named public_html, www, or htdocs.
  3. Upload everything inside the build folder into that public folder (upload the contents, not the build folder itself).
  4. Make sure index.html sits directly in the public folder.
  5. Visit your domain in a browser — the site should appear.
If the site will live in a sub-folder (e.g. yourdomain.com/sakura/) rather than the main domain, it must be rebuilt with that path set. This is a small developer task — see section 4 / contact us.

4. Make it fully live: accounts & payments Developer

In demo mode, sign-ups and payments are simulated. To use real member accounts, a database and real Stripe payments, connect the two services below. Full technical steps are in source/docs/backend-setup.md. Overview:

A. Member accounts & database — Supabase (free to start)

  • Create a project at supabase.com (this account/billing stays yours).
  • Open the project's “SQL Editor” and run the file source/supabase/schema.sql (copy‑paste & Run).
  • Turn on Email sign-in under “Authentication”. For reliable sign-up emails, add an email sender (e.g. Resend/SendGrid) — optional but recommended.
  • From “Project Settings → API”, copy the Project URL and the anon public key.

B. Payments — Stripe (no server needed)

  • In your Stripe dashboard, create a Payment Link for each plan (Monthly / 3-Month / Yearly / Lifetime).
  • For each link, set the after-payment redirect to https://your-site/subscribe/?checkout=success.
  • Copy each Payment Link URL.

C. Enter the values and rebuild

  • In source/, copy .env.local.example to .env.local and paste the Supabase URL/key and the Stripe links into it.
  • Rebuild: in source/ run npm install then npm run build, and re-publish the new out/ folder (same as section 2/3).
  • The same screens now use real accounts, database and payments — no other changes.

Optional: live Anime Learning videos (YouTube)

  • At console.cloud.google.com: create a project → enable YouTube Data API v3 → create an API key.
  • Restrict the key: HTTP referrers = your domain, API = YouTube Data API only (it's used in the browser).
  • Set NEXT_PUBLIC_YOUTUBE_API_KEY in .env.local and rebuild.
  • Free — 10,000 units/day (~90 searches), no credit card. Without a key, sample videos + “Open on YouTube” are shown.
This stage needs a developer (it involves a few commands). If you'd like, we can do the full connection and go-live for you — just ask.

5. Preview on your computer & demo logins

Preview locally (optional, needs Node.js)

  1. Install Node.js (LTS) from nodejs.org.
  2. Open Terminal / Command Prompt in the source folder.
  3. Run npm install, then npm run dev.
  4. Open http://localhost:3000 in your browser.

Demo logins (demo mode)

  • Admin (to see the admin console): admin@sakura.dev / password
  • Or click “Sign up” to make an account, or “Continue as guest”.
