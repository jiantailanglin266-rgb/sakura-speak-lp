1. 納品物の内容

ZIPファイル1つ（Sakura Speak 納品一式 2026-06-30.zip）をお渡ししています。まず解凍してください （Windowsは右クリック→「すべて展開」、Macはダブルクリック）。中身は次の通りです。

フォルダ / ファイル内容
build/完成したWebサイトのファイル — このまま公開できます。これをネットに上げます。
source/ソースコード一式（開発者向け／今後の改修用）。
docs/PDF：提案書・見積書・開発レポート。
HANDOVER.md技術的な概要メモ。
全体像：このサイトは2段階で考えるとシンプルです。第1段階（本書 2〜3章）＝サイトをネットに公開し、 「デモモード」で見て触れる状態にする。第2段階（4章）＝実際のアカウント・決済を接続する。 第1段階はプログラミング不要、第2段階は開発者と進めるのがおすすめです（当社でも対応可能）。

2. いちばん簡単な公開方法 コード不要

無料サービス Netlify Drop を使い、最短で公開リンクを発行します。

  1. 納品ZIPを解凍し（1章）、build フォルダを見つけます。
  2. ブラウザで https://app.netlify.com/drop を開きます。
  3. build フォルダごと、ページ上（「Drag and drop your site folder here」と書かれた領域）にドラッグ＆ドロップします。
  4. 数秒待つと https://〇〇〇.netlify.app のような公開リンクが表示されます。これで公開完了です 🎉
  5. （任意）無料のNetlifyアカウントを作ると、サイトの保持やリンク名の変更ができます。
  6. （任意）独自ドメイン接続：Netlifyの「Domain settings」→「Add a domain」から画面の案内に従います。
ヒント：他の無料ホスティング（Cloudflare Pages、Vercel、GitHub Pages 等）でも手順は同じです。 どれを使う場合も、やることはbuild フォルダの中身を公開するだけです。
ご注意：build/index.html をパソコン上でそのままダブルクリックして開かないでください （デザインやリンクが正しく表示されません）。必ず上記の手順でホスティングに公開するか、5章のローカル確認をご利用ください。

3. 自社サーバー／レンタルサーバーに置く コード不要

既にドメイン・サーバー（cPanel、FTPアカウント等）をお持ちの場合：

  1. サーバーの管理画面にログイン（またはFTPソフト、例：FileZilla を開く）。
  2. 公開フォルダを開きます。多くは public_html / www / htdocs という名前です。
  3. build フォルダの中身すべてを、その公開フォルダにアップロードします（build フォルダ自体ではなく、中身を）。
  4. index.html が公開フォルダの直下に来ているか確認します。
  5. ブラウザでドメインにアクセスすると、サイトが表示されます。
サブフォルダで公開する場合（例：yourdomain.com/sakura/）は、そのパスを設定して再ビルドが必要です。 これは小さな開発作業です（4章／当社にご相談ください）。

4. 本番化：アカウントと決済の接続 開発者向け

デモモードでは登録・決済は擬似動作です。実際の会員アカウント・データベース・Stripe決済を使うには、 下記2サービスを接続します。詳細手順は source/docs/backend-setup.md にあります。概要：

A. 会員アカウント＆データベース — Supabase（無料から）

  • supabase.com でプロジェクトを作成（このアカウント・課金は貴社名義のままです）。
  • 「SQL Editor」を開き、source/supabase/schema.sql の内容を貼り付けて実行します。
  • 「Authentication」でメールログインを有効化。確実な登録メール配信には送信サービス（Resend/SendGrid 等）の追加を推奨（任意）。
  • 「Project Settings → API」から Project URL と anon public key をコピーします。

B. 決済 — Stripe（サーバー不要）

  • Stripeダッシュボードで、各プラン（月額／3ヶ月／年額／買い切り）の Payment Link を作成します。
  • 各リンクの「支払い後リダイレクト先」を https://公開URL/subscribe/?checkout=success に設定します。
  • 各Payment LinkのURLをコピーします。

C. 値を入力して再ビルド

  • source/ 内の .env.local.example を .env.local にコピーし、SupabaseのURL/キーとStripeリンクを貼り付けます。
  • 再ビルド：source/ で npm install → npm run build を実行し、新しい out/ を再公開します（2章／3章と同じ要領）。
  • 同じ画面が、実アカウント・データベース・決済で動作するようになります（他の変更は不要）。

任意：Anime Learning の実動画（YouTube）

  • console.cloud.google.com でプロジェクト作成 → YouTube Data API v3 を有効化 → APIキーを作成。
  • キーを制限：HTTPリファラー＝自社ドメイン、API＝YouTube Data APIのみ（ブラウザで使うため）。
  • .env.local に NEXT_PUBLIC_YOUTUBE_API_KEY を設定して再ビルド。
  • 無料 — 1日1万ユニット（約90回検索）・カード不要。キーが無くてもサンプル動画＋「Open on YouTube」で公式動画に到達できます。
この段階はコマンド操作を含むため、開発者の対応が必要です。ご希望であれば、接続〜本番公開まで当社で代行いたします。お気軽にどうぞ。

5. パソコンでの確認＆デモ用ログイン

ローカルで確認する（任意・Node.js が必要）

  1. nodejs.org から Node.js（LTS版）をインストール。
  2. source フォルダでターミナル／コマンドプロンプトを開きます。
  3. npm install → npm run dev を実行。
  4. ブラウザで http://localhost:3000 を開きます。

デモ用ログイン（デモモード）

  • 管理者（管理画面の確認用）：admin@sakura.dev / password
  • または「Sign up」で登録、「Continue as guest」でゲスト入室。

6. 困ったときは

症状対処
index.html を開いたらデザインが崩れるパソコンから直接開かないでください。build をホスティングに公開（2章）するか、ローカル確認（5章）を使います。
公開したが画像やフォントが出ないbuild/ の中身を、index.html と _next フォルダが同じ場所に来るようアップロードしてください。
登録・決済が保存されないデモモードでは正常です。4章でSupabase／Stripeを接続してください。
サブフォルダでリンクが404になるサブパスを設定して再ビルドが必要です（小さな開発作業／4章・当社へご相談）。
